#### Exploratory Data Analysis


- Telecom Company Churn Predictions 📞
  - data comes from IBM and can be found on [kaggle](https://www.kaggle.com/blastchar/telco-customer-churn/kernels)
  - [Jupyter notebook](https://github.com/leblanchannah/DataMining873/blob/master/873groupAset1churn/churn.ipynb)
  - results: Used customer data to predict which customers were likely to leave (churn) the telecom company. Based on visualizations, issues with fibre optic services and month-to-month contracts seem to make customers leave the company. Using SVM with a linear kernel, feature importance values indicate that paperless billing is important for predicting which customers will stay. 
  
  
- California Crime Rates 👮‍♀️
  - classification problem, predicting crime rate ranges based on features of California counties
  - [GitHub Repo](https://github.com/leblanchannah/DataMining873/tree/master/873groupAset2crime)
  - results: Overlap between the robbery, property, burglary, larceny and motor vehicle theft classes and missing data makes this problem especially difficult. Features within the income and benefits category seem to contribute the most to crime rates. Counties with higher average number of people living within a household and low vacancies have lower crime rates. Information on the amount of crimes that are actually solved/ cases closed would also be intersting. 
  
  
- Fraud Detection 🕵️‍♂️
  - dataset from
Dechow, Patricia M. and Ge, Weili and Larson, Chad Russell and Sloan, Richard G.,[Predicting Material Accounting Misstatements](https://www.thecaq.org/sites/default/files/DECHOW_et_al-2011-Contemporary_Accounting_Research.pdf) (April 21, Paper.2010). Contemporary Accounting Research, Forthcoming; AAA 2008 Financial Accounting and Reporting Section (FARS) 
  - [GitHub Repo](https://github.com/leblanchannah/Fraud-Detection-SdA)
  - results: The abstract features in this dataset makes it difficult to use real world experiences to interpret the graphs generated. There is a lot of data missing which ends up being useful, it seems like fraudulent companies try to obscure information. The addition of features that indicate which values are missing for a given company are helpful in this situation. For this dataset I was given the task to cluster the data using a stacked denoising autoencoder implemented with Keras. I compressed the data into a three dimensional and two dimensional latent layers and graphed the results.
